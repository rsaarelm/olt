Otlbook turns VimOutliner files into static wiki webpages https://github.com/vimoutliner/vimoutliner Write regular outline documents Use WikiWord-s to name important concepts Define them either inline WikiWord WikiWord-s are the original way to create wiki structure They're two or more words written in SnakeCase OtlBook allows numbers in them: Test1234Word Or in separate files, put `WikiWord.otl.html` somewhere in your project folder Run the otlbook tool to generate a tags file and a custom javascript file ; ./otlbook.py The javascript file will contain link information of all your local wiki links New syntax Mark important nodes with trailing asterisk * Add inline images with markdown-style `![alt txt](image path)` ![icon](icon-journal.png) Tips Make git's pre-commit hook regenerate tags ; cat > .git/hooks/pre-commit <<EOF ; #!/bin/sh ; ./otlbook.py ; git add votl.js ; EOF ; chmod a+x .git/hooks/pre-commit The HTML layer hack isn't quite perfect Things in text that look like they are opening a HTML tag, `<likethis` and not closing it afterwards will break the browser's parser Alphabetic characters immediately after the < are the problem You need to use `&lt;` in place of the `<` for them Make Vim fill the preformatted otl block prefix VimOutliner enables prefix filling by marking prefixes as comments Tag the preformatted block prefix as comment ; :se comments+=:< Disable autowrapping in comment text ; :se formatoptions-=c
